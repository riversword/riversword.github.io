流氓电信

发表于 2018-05-21 |
字数统计:

好久没写博客了，最近给博客换了新主题，打算写几篇博文归纳下知识。结果博客上线没两天，突然打不开了，页面显示如下：

  • 报错大意是 $ 未定义，最先想到的是 jQuery 引用路径出错了，但是没道理啊，最近也没更新提交。

于是，hexo clean 清理文件后，重新部署。但仍然出现相同的报错。

  • 报错信息中有一个是说，地址为 http://183.59.53.197:3737 的请求被客户端拦截。貌似 hexo 中引用文件 url 没有是以 ip 开头的。

先看看这货链接到的文件究竟是啥。

搜索看看这货到底是是什么玩意：

真相大白，惊不惊喜，意不意外。⊙﹏⊙‖∣° ，以前只是有听说电信很流氓，没想到还真是流氓。

这个流氓电信还是个厉害角色，要打倒它还不太容易。1.按下面那样屏蔽链接（有时候会不管用）；2.需要关闭浏览器的 AdBlock 插件，前面报错信息中说的 http://183.59.53.197:3737 ERR BLOCKED BY CLIENT 应该就是指被 AdBlock 插件拦截了。

没办法，整栋公寓的宽带被电信给包揽了。如果可以有别的选择，我是绝不会选择流氓电信的，好憋屈啊 ( T___T )。

比较奇怪的是，就单单博客这个网址打开时，会出现以上情况，浏览其他网页并不会，在 sublime 中检索 hexo 的部署文件，也没发现其中有包含 http://183.59.53.197:3737 的代码；在 QQ 浏览器（未装任何插件）中打开博客页面时，页面也是会加载不出，并且会直接跳出电信的两个广告页面。
